# 日志管理命令精讲

日志是 Linux 系统运维的“眼睛”。当系统出现问题时，日志是排查问题的第一手资料。这组命令帮助你查看、过滤、监控和管理系统及应用程序的日志。

---

## 一、核心概念

### 日志系统演进
| 时代 | 日志系统 | 特点 |
|------|----------|------|
| **传统** | `/var/log/*` | 分散的文本文件 |
| **现代** | `systemd-journald` | 集中式日志服务（`journalctl`） |

### Manjaro 的日志架构
Manjaro 同时支持两种方式：
1. **`journalctl`**：systemd 的统一日志管理（**推荐**）
2. **`/var/log/`**：传统日志文件（如 `syslog`、`kern.log`）

### 日志级别（重要性从高到低）
| 级别 | 含义 | 说明 |
|------|------|------|
| `emerg` | 紧急 | 系统不可用 |
| `alert` | 警报 | 需要立即处理 |
| `crit` | 严重 | 关键条件 |
| `err` | 错误 | 错误条件 |
| `warning` | 警告 | 警告信息 |
| `notice` | 通知 | 重要信息 |
| `info` | 信息 | 一般信息 |
| `debug` | 调试 | 调试信息 |

## 2. `journalctl -xe` — 查看系统日志（最常用）

### 命令格式
```bash
journalctl -xe              # 查看系统日志（带解释）
journalctl -xe -n 50        # 只显示最近50行
journalctl -xe --since "1 hour ago"  # 最近1小时
```

### 参数说明
| 参数 | 含义 |
|------|------|
| `-x` | 显示日志的详细解释（**推荐**） |
| `-e` | 跳转到日志末尾（**推荐**） |
| `-n N` | 显示最近 N 行 |
| `-f` | 实时跟踪（类似 `tail -f`） |
| `-u 服务` | 只显示特定服务的日志 |

### 输出示例
```
-- Logs begin at Mon 2026-06-22 10:00:00 CST, end at Mon 2026-06-22 15:30:00 CST. --
Jun 22 14:30:15 nuoxi-system dockerd[1234]: time="2026-06-22T14:30:15+08:00" level=info msg="API listen on /var/run/docker.sock"
Jun 22 14:30:20 nuoxi-system systemd[1]: Started Docker Application Container Engine.
-- Subject: Unit docker.service has finished start-up
-- Defined-By: systemd
-- Support: https://wiki.archlinux.org/title/Systemd
-- 
-- Unit docker.service has finished starting up.
```

### 实际应用
```bash
# 查看系统日志（你排查问题时最常用的命令）
journalctl -xe

# 查看最近 20 行错误日志
journalctl -xe -n 20

# 查看最近 1 小时的日志
journalctl --since "1 hour ago"

# 查看今天的日志
journalctl --since "today"
```

## 3. `journalctl -f` — 实时日志

### 命令格式
```bash
journalctl -f        # 实时跟踪系统日志
journalctl -u docker -f   # 实时跟踪特定服务
```

### 类似 `tail -f`
`journalctl -f` 相当于 `tail -f /var/log/syslog`，但功能更强大。

### 实际应用
```bash
# 实时查看系统日志（监控系统状态）
journalctl -f

# 实时查看 Docker 日志（你经常用）
journalctl -u docker -f

# 实时查看 AstrBot 日志
journalctl --user -u astrbot -f
```

---

## 4. `journalctl -u 服务` — 查看特定服务日志

### 命令格式
```bash
journalctl -u 服务名         # 查看服务日志
journalctl -u docker -f      # 实时查看 Docker 日志
journalctl -u docker -n 50   # 最近50行
```

### 实际应用
```bash
# 查看 Docker 日志
journalctl -u docker

# 查看 Docker 实时日志（你经常用）
journalctl -u docker -f

# 查看 1Panel 日志
sudo journalctl -u 1panel -f

# 查看 AstrBot 日志（用户服务）
journalctl --user -u astrbot -f

# 查看 NetworkManager 日志
journalctl -u NetworkManager
```

## 5. `journalctl --since` — 查看指定时间日志

### 命令格式
```bash
journalctl --since "时间"              # 指定开始时间
journalctl --until "时间"              # 指定结束时间
journalctl --since "1 hour ago"        # 最近1小时
journalctl --since "yesterday"         # 从昨天开始
journalctl --since "2026-06-22 10:00:00"  # 指定时间
```

### 时间格式示例
| 格式 | 示例 | 说明 |
|------|------|------|
| 相对时间 | `--since "2 hours ago"` | 2小时前 |
| 相对时间 | `--since "yesterday"` | 昨天 |
| 相对时间 | `--since "today"` | 今天开始 |
| 绝对时间 | `--since "2026-06-22 10:00:00"` | 指定时间 |
| 绝对时间 | `--since "2026-06-22"` | 指定日期 |

### 实际应用
```bash
# 查看最近2小时的日志
journalctl --since "2 hours ago"

# 查看今天的日志
journalctl --since "today"

# 查看昨天到现在的日志
journalctl --since "yesterday"

# 查看指定时间段的日志
journalctl --since "2026-06-22 10:00:00" --until "2026-06-22 11:00:00"
```

## 6. `journalctl -n 50` — 查看最近日志

### 命令格式
```bash
journalctl -n 50         # 最近50行
journalctl -n 100        # 最近100行
journalctl -u docker -n 50  # 特定服务最近50行
```

### 实际应用
```bash
# 查看最近50行
journalctl -n 50

# 查看 Docker 最近30行
journalctl -u docker -n 30

# 查看最近的错误
journalctl -n 100 | grep -i error
```

---

## 7. `journalctl --vacuum-size=500M` — 限制日志总大小

### 命令格式
```bash
sudo journalctl --vacuum-size=500M    # 限制日志总大小
sudo journalctl --vacuum-time=7d      # 保留7天日志
```

### 实际应用（你之前执行过）
```bash
# 将日志总大小限制在 500MB
sudo journalctl --vacuum-size=500M

# 只保留最近7天的日志
sudo journalctl --vacuum-time=7d

# 查看当前日志占用
du -sh /var/log/journal/
```

### 为什么需要清理？
系统日志可能会占用大量磁盘空间，特别是长时间运行的系统。定期清理可以释放空间。

## 8. `dmesg -T` — 查看内核日志（带时间戳）

### 命令格式
```bash
dmesg -T              # 查看内核日志（人类可读时间）
dmesg -T | tail -20   # 最近20行
dmesg -w              # 实时监控
```

### 输出示例
```bash
dmesg -T | tail -5
```
```
[Mon Jun 22 14:30:15 2026] usb 2-1: new high-speed USB device number 4 using xhci_hcd
[Mon Jun 22 14:30:16 2026] usb 2-1: New USB device found, idVendor=0bda, idProduct=8152
[Mon Jun 22 14:30:16 2026] Docker daemon started: API listen on /var/run/docker.sock
```

### `dmesg` 常见用途
| 场景 | 命令 |
|------|------|
| 查看 USB 设备 | `dmesg -T | grep -i usb` |
| 查看磁盘信息 | `dmesg -T | grep -i disk` |
| 查看硬件错误 | `dmesg -T | grep -i error` |
| 实时监控 | `dmesg -w` |

---

## 9. `dmesg -w` — 实时内核日志

### 命令格式
```bash
dmesg -w             # 实时监控内核日志
```

### 实际应用
```bash
# 实时监控内核事件（插入 U 盘时能看到）
dmesg -w

# 同时过滤错误
dmesg -w | grep -i error
```

## 10. `tail -f /var/log/日志文件` — 查看传统日志文件

### 命令格式
```bash
sudo tail -f /var/log/syslog     # 系统日志
sudo tail -f /var/log/kern.log   # 内核日志
sudo tail -f /var/log/auth.log   # 认证日志
```

### 传统日志文件位置
| 文件 | 内容 |
|------|------|
| `/var/log/syslog` | 系统综合日志 |
| `/var/log/kern.log` | 内核日志 |
| `/var/log/auth.log` | 认证/安全日志 |
| `/var/log/dmesg` | 启动日志 |
| `/var/log/Xorg.0.log` | X11 图形日志 |

### 实际应用
```bash
# 查看系统日志
sudo tail -f /var/log/syslog

# 查看认证日志（登录失败）
sudo tail -f /var/log/auth.log

# 查看图形界面日志
tail -f /var/log/Xorg.0.log
```

## 11. `grep -i 关键词 /var/log/*` — 搜索日志

### 命令格式
```bash
sudo grep -i "关键词" /var/log/*           # 搜索所有日志
sudo grep -i "error" /var/log/syslog      # 搜索特定文件
sudo grep -i -r "关键词" /var/log/        # 递归搜索
```

### 参数说明
| 参数 | 含义 |
|------|------|
| `-i` | 忽略大小写（**常用**） |
| `-r` | 递归搜索 |
| `-l` | 只显示文件名 |
| `-A 5` | 显示匹配行及后5行 |
| `-B 5` | 显示匹配行及前5行 |

### 实际应用
```bash
# 搜索所有错误
sudo grep -i error /var/log/syslog

# 搜索最近100行中的错误
sudo tail -100 /var/log/syslog | grep -i error

# 搜索所有日志文件中的 "fail"
sudo grep -i -r "fail" /var/log/

# 显示错误及前后5行
sudo grep -i -A 5 -B 5 "error" /var/log/syslog
```

## 💡 你的实战场景

### 场景1：排查 AstrBot 启动问题
```bash
# 查看 AstrBot 实时日志
journalctl --user -u astrbot -f

# 查看最近 50 行
journalctl --user -u astrbot -n 50

# 查看今天的 AstrBot 日志
journalctl --user -u astrbot --since "today"
```

### 场景2：检查 Docker 服务是否正常
```bash
# 查看 Docker 实时日志
journalctl -u docker -f

# 查看 Docker 最近错误
journalctl -u docker -n 50 | grep -i error
```

### 场景3：系统变慢时排查
```bash
# 查看系统错误
journalctl -xe | grep -i error

# 查看最近 100 行
journalctl -n 100

# 查看最近 5 分钟
journalctl --since "5 minutes ago"
```

### 场景4：磁盘空间不足，清理日志
```bash
# 查看日志占用
du -sh /var/log/journal/

# 限制日志大小为 500MB
sudo journalctl --vacuum-size=500M

# 验证释放的空间
du -sh /var/log/journal/
```

---

## ⚠️ 注意事项

1. **`journalctl` 是 systemd 的日志工具**，在 Manjaro 中完全可用。
2. **`/var/log/` 中的日志文件可能不存在**，取决于系统配置。
3. **`journalctl` 日志是持久化存储的**，重启后依然存在。
4. **清理日志需要 `sudo` 权限**。
5. **`dmesg` 日志是内核环缓存**，重启后会清空。
6. **日志文件可能占用大量磁盘空间**，建议定期清理。
7. **`journalctl -xe` 是排查问题时最常用的命令**，建议记住。
