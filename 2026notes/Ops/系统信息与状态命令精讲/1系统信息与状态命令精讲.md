这组命令是 Linux 系统管理的“入门必修课”，帮你快速了解系统的基本情况。下面逐一深入讲解：

---

## 1. `uname -a` — 查看内核版本和系统信息

### 命令格式
```bash
uname -a
```

### 输出示例
```
Linux nuoxi-system 6.18.33-1-MANJARO #1 SMP PREEMPT_DYNAMIC Thu May 21 07:23:17 UTC 2026 x86_64 GNU/Linux
```

### 各部分含义
| 字段 | 含义 | 示例值 |
|------|------|--------|
| `Linux` | 操作系统名称 | Linux |
| `nuoxi-system` | **主机名**（网络标识） | 你的计算机名 |
| `6.18.33-1-MANJARO` | **内核版本**（核心信息） | 当前内核版本号 |
| `#1 SMP...` | 编译信息 | 编译序号和时间 |
| `x86_64` | **硬件架构**（CPU 位数） | 64位系统 |
| `GNU/Linux` | 操作系统类型 | 基于 GNU 的 Linux |

### 常用变体
| 参数 | 含义 | 示例输出 |
|------|------|----------|
| `uname -r` | 只显示内核版本 | `6.18.33-1-MANJARO` |
| `uname -m` | 只显示硬件架构 | `x86_64` |
| `uname -n` | 只显示主机名 | `nuoxi-system` |
| `uname -s` | 只显示内核名 | `Linux` |
| `uname -v` | 只显示内核版本详细信息 | `#1 SMP PREEMPT_DYNAMIC...` |

### 实际应用
- **排查内核问题**：检查内核版本是否与某软件兼容
- **确认系统架构**：下载软件时确认是 `x86_64` 还是 `aarch64`
- **快速系统摘要**：新手入门时了解自己的系统信息

## 2. `hostname` — 查看/设置主机名

### 基本用法
```bash
hostname                    # 查看当前主机名
hostnamectl set-hostname 新名字   # 永久修改主机名
```

### 输出示例（普通查看）
```
nuoxi-system
```

### 进阶用法
| 命令 | 用途 | 示例 |
|------|------|------|
| `hostname -f` | 显示完全限定域名（FQDN） | `nuoxi-system.localdomain` |
| `hostname -s` | 只显示短主机名（第一个点之前） | `nuoxi-system` |
| `hostname -I` | 显示所有 IP 地址（**你的常用命令，注意大小写**） | `172.30.174.180` |
| `hostname -i` | 显示 IP 地址（可能返回环回地址） | `127.0.0.1` |

> **注意**：`hostname -I` 在 Manjaro 默认的 `hostname` 简化版本中不支持，你已经遇到过这个问题，之后改用 `ip addr` 来查看 IP。

### 主机名分类
| 类型 | 存储位置 | 作用范围 | 持久性 |
|------|----------|----------|--------|
| 静态主机名 | `/etc/hostname` | 整个系统 | 永久（重启后生效） |
| 临时主机名 | 内存中 | 当前会话 | 临时（重启后失效） |
| 漂亮主机名 | `/etc/machine-info` | 显示用（如 GNOME 界面） | 永久 |

### 手动修改（永久）
```bash
# 方法一：使用 hostnamectl（推荐）
sudo hostnamectl set-hostname my-server

# 方法二：直接编辑文件
sudo nano /etc/hostname
# 写入新主机名，保存退出

# 方法三：使用 sysctl
sudo sysctl kernel.hostname=my-server
```

### 实际应用
- **网络标识**：局域网内识别你的机器
- **个性化**：给虚拟机起一个有意义的名字（如 `manjaro-dev`）
- **自动化脚本**：在脚本中获取主机名用于日志记录

## 3. `uptime` — 查看系统运行时间和负载

### 命令格式
```bash
uptime
```

### 输出示例
```
 14:52:31 up 2 days,  3:14,  2 users,  load average: 0.45, 0.32, 0.28
```

### 各部分含义
| 字段 | 含义 | 示例值 |
|------|------|--------|
| `14:52:31` | 当前系统时间 | 下午 14:52 |
| `up 2 days, 3:14` | **运行时间**：已运行 2天3小时14分 | 系统持续运行时长 |
| `2 users` | 当前登录用户数 | 2个用户在线 |
| `load average:` | **系统负载**（CPU 使用情况） | 三个时间点的平均值 |
| `0.45, 0.32, 0.28` | **1分钟/5分钟/15分钟** 平均负载 | 数字越小越空闲 |

### 负载值含义（Load Average）
| 负载值 | 含义 | 对应你的系统 |
|--------|------|--------------|
| 0.00 ~ 0.50 | 空闲状态，系统很轻松 | ✅ 正常状态 |
| 0.50 ~ 1.00 | 有一定负载，但正常 | ✅ 一般状态 |
| 1.00 ~ 2.00 | 接近满载，可能有点慢 | ⚠️ 需要注意 |
| > 2.00 | 严重过载，系统明显变慢 | ❌ 需要排查 |

> **负载值判断公式**：负载值 / CPU 核心数 > 1 时系统过载。
> 你的系统是 `x86_64` 多核（你的是 16 核），平均负载 1.0 以下都很健康。

### 实际应用
- **判断系统是否卡死**：如果负载持续很高，说明系统资源紧张
- **排查性能问题**：结合 `top` 或 `htop` 找出高负载进程
- **监控系统健康**：写入监控脚本，定期检测负载

### 组合命令：快速查看系统状态
```bash
# 一键查看系统基本状态（你经常用的组合）
uptime && free -h && df -h
```

## 4. `dmesg` — 查看内核日志

### 命令格式
```bash
dmesg                    # 查看所有内核日志
dmesg | tail -20         # 查看最近20行
```

### 输出示例（尾部）
```
[ 3567.123456] usb 2-1: new high-speed USB device number 4 using xhci_hcd
[ 3567.456789] usb 2-1: New USB device found, idVendor=0bda, idProduct=8152
[ 3567.456790] usb 2-1: New USB device strings: Mfr=1, Product=2, SerialNumber=3
[ 3580.000001] Docker daemon started: API listen on /var/run/docker.sock
```

### 常用参数
| 参数 | 用途 | 示例 |
|------|------|------|
| `dmesg -T` | **显示人类可读的时间戳**（推荐） | `dmesg -T \| tail -20` |
| `dmesg -w` | 实时监控内核日志（类似 `tail -f`） | `dmesg -w` |
| `dmesg -l err` | 只显示错误级别（`err`, `warn`, `info`） | `dmesg -l err` |
| `dmesg -H` | 人类可读格式（带时间） | `dmesg -H` |
| `dmesg | grep -i error` | 搜索错误信息 | `dmesg \| grep -i error` |
| `dmesg | grep -i usb` | 搜索 USB 设备信息 | `dmesg \| grep -i usb` |

### 实际应用
- **USB 设备问题**：插入 U 盘没有反应时查看 USB 检测日志
- **硬件故障排查**：查看是否有硬件报错（如磁盘、内存问题）
- **系统启动问题**：查看内核启动过程中的报错

## 5. `lsb_release -a` — 查看发行版信息

### 命令格式
```bash
lsb_release -a
```

### 输出示例
```
LSB Version:    n/a
Distributor ID: ManjaroLinux
Description:    Manjaro Linux
Release:        24.2.0
Codename:       Orion
```

### 可能遇到的问题
- **命令不存在**：如果提示 `command not found`，需要安装：
  ```bash
  sudo pacman -S lsb-release
  ```

### 常用参数
| 参数 | 用途 | 示例输出 |
|------|------|----------|
| `lsb_release -d` | 只显示描述 | `Description: Manjaro Linux` |
| `lsb_release -r` | 只显示版本号 | `Release: 24.2.0` |
| `lsb_release -c` | 只显示代号 | `Codename: Orion` |
| `lsb_release -i` | 只显示发行商 ID | `Distributor ID: ManjaroLinux` |
| `lsb_release -a` | 显示所有信息（**最常用**） | 全部输出 |

### 实际应用
- **确认系统版本**：在安装软件或查阅文档时确认系统版本
- **兼容性检查**：检查软件是否支持当前发行版
- **自动化脚本**：在脚本中根据发行版执行不同的命令

## 6. `cat /etc/os-release` — 查看系统版本（更通用）

### 命令格式
```bash
cat /etc/os-release
```

### 输出示例
```
NAME="Manjaro Linux"
ID=manjaro
ID_LIKE=arch
PRETTY_NAME="Manjaro Linux"
ANSI_COLOR="1;32"
HOME_URL="https://manjaro.org/"
DOCUMENTATION_URL="https://wiki.manjaro.org/"
SUPPORT_URL="https://manjaro.org/forum/"
BUG_REPORT_URL="https://docs.manjaro.org/"
PRIVACY_POLICY_URL="https://manjaro.org/privacy-policy/"
LOGO=manjaro
```

### 关键字段含义
| 字段 | 含义 | 示例值 |
|------|------|--------|
| `NAME` | 操作系统名称 | `Manjaro Linux` |
| `ID` | 系统标识（用于脚本判断） | `manjaro` |
| `PRETTY_NAME` | 友好显示名称 | `Manjaro Linux` |
| `VERSION_ID` | 版本号（可能显示） | `24.2.0` |
| `HOME_URL` | 官网地址 | `https://manjaro.org/` |

### 为什么 `os-release` 比 `lsb_release` 更推荐？
| 对比项 | `/etc/os-release` | `lsb_release` |
|--------|-------------------|---------------|
| **默认安装** | ✅ 所有 Linux 系统都有 | ❌ 需要额外安装 |
| **信息完整性** | ✅ 包含更多字段 | ⚠️ 部分信息可能缺失 |
| **脚本兼容性** | ✅ 格式统一 | ⚠️ 不同发行版输出格式略有差异 |

### 实际应用
- **脚本判断系统**：在脚本中检测发行版并执行对应操作
  ```bash
  if grep -q "manjaro" /etc/os-release; then
      echo "This is Manjaro!"
  fi
  ```
- **快速确认系统**：不需要记忆 `lsb_release` 是否已安装

## 📊 六命令速查对比表

| 命令 | 核心信息 | 输出速度 | 常用场景 |
|------|----------|----------|----------|
| `uname -a` | 内核版本 + 架构 | 极快 | 确认内核版本 |
| `hostname` | 主机名 | 极快 | 查看/修改计算机名 |
| `uptime` | 运行时间 + 负载 | 极快 | 快速判断系统负载 |
| `dmesg \| tail -20` | 最近内核日志 | 较快 | 排查硬件/驱动问题 |
| `lsb_release -a` | 发行版信息 | 较快 | 确认发行版和版本 |
| `cat /etc/os-release` | 发行版详细信息 | 极快 | 脚本中判断系统类型 |

## 💡 日常运维组合命令

```bash
# 系统状态总览（你的常用组合）
echo "=== 系统信息 ===" && uname -a
echo "=== 主机名 ===" && hostname
echo "=== 运行状态 ===" && uptime
echo "=== 发行版 ===" && cat /etc/os-release
echo "=== 内存 ===" && free -h
echo "=== 磁盘 ===" && df -h
```

这条组合命令能让你在几秒钟内获得系统的全貌信息。
