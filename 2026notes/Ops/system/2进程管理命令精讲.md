这组命令是 Linux 系统管理的“手术刀”——让你能够查看、控制、调整系统中所有正在运行的程序。下面逐一深入讲解：

---

## 1. `ps aux` — 查看所有进程（最常用）

### 命令格式
```bash
ps aux                    # 查看所有进程
ps aux | grep astrbot     # 查找特定进程（你经常用）
```

### 输出示例
```
USER       PID  %CPU  %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
nuoxi     1234  0.5   2.1  123456  45678 ?        Ssl  10:30   0:15 /usr/bin/python3 /usr/bin/astrbot
root      5678  0.0   0.1  234567  12345 ?        Ss   10:20   0:00 /usr/bin/dockerd
nuoxi     9012  0.1   0.5  98765  23456 pts/0    S+   10:35   0:02 htop
```

### 各部分含义
| 字段 | 含义 | 说明 |
|------|------|------|
| `USER` | 进程所有者 | 哪个用户启动的 |
| `PID` | **进程 ID**（唯一标识） | 最重要的字段，用于 kill、renice 等操作 |
| `%CPU` | **CPU 使用率** | 占 CPU 时间的百分比 |
| `%MEM` | **内存使用率** | 占物理内存的百分比 |
| `VSZ` | 虚拟内存大小 | 进程申请的总虚拟内存（KB） |
| `RSS` | 实际物理内存 | 进程占用的物理内存（KB） |
| `TTY` | 终端类型 | `?` = 后台进程，`pts/0` = 终端 |
| `STAT` | **进程状态** | 见下表 |
| `START` | 启动时间 | 进程开始运行的时间 |
| `TIME` | CPU 累计时间 | 进程占用的 CPU 总时间 |
| `COMMAND` | **命令名称** | 完整的命令行（含参数） |

### 进程状态（STAT）含义
| 状态码 | 含义 | 说明 |
|--------|------|------|
| `R` | 运行中 (Running) | 正在执行或在运行队列中 |
| `S` | 睡眠 (Sleeping) | 等待某个事件（可中断） |
| `D` | 不可中断睡眠 | 等待 I/O（如磁盘读写） |
| `Z` | 僵尸 (Zombie) | 已结束但父进程未回收 |
| `T` | 已停止 (Stopped) | 被信号暂停（如 `Ctrl+Z`） |
| `s` | 会话领导者 | 该进程是会话的首进程 |
| `+` | 前台进程 | 在前台运行 |
| `l` | 多线程进程 | 使用多线程 |

### `ps aux` vs `ps -ef` 区别
| 对比项 | `ps aux` | `ps -ef` |
|--------|----------|----------|
| 格式风格 | BSD 风格（更直观） | 标准 System V 风格 |
| 显示内容 | 包含 %CPU、%MEM | 包含 PPID（父进程 ID） |
| 推荐度 | ✅ 更常用 | ⚠️ 用于需要父进程信息时 |

### 实际应用
```bash
# 查找 AstrBot 进程
ps aux | grep astrbot | grep -v grep

# 按内存使用排序（你之前用过）
ps aux --sort=-%mem | head -10

# 按 CPU 使用排序
ps aux --sort=-%cpu | head -10

# 查看特定用户的进程
ps aux | grep "^nuoxi"
```

## 2. `ps -ef` — 查看进程（标准格式，含父进程）

### 命令格式
```bash
ps -ef
```

### 输出示例
```
UID        PID  PPID  C STIME TTY          TIME CMD
nuoxi     1234  5678  0 10:30 ?        00:00:15 python3 /usr/bin/astrbot
root      5678  2345  0 10:20 ?        00:00:00 /usr/bin/dockerd
```

### 额外字段（相比 `ps aux`）
| 字段 | 含义 | 说明 |
|------|------|------|
| `PPID` | **父进程 ID** | 哪个进程启动了这个进程 |
| `C` | CPU 使用率 | 与 `%CPU` 类似 |
| `STIME` | 启动时间 | 进程启动的时间点 |

### 实际应用
```bash
# 查看进程树（父子关系）
ps -ef | grep astrbot
# 输出显示 PPID，可以看到是谁启动了 AstrBot

# 结合 `pstree` 查看完整进程树
pstree -p | grep astrbot
```

## 3. `top` / `htop` — 实时进程监控

### `top` — 传统实时监控
```bash
top
```

### 输出示例（上半部分）
```
top - 14:52:31 up 2 days,  3:14,  2 users,  load average: 0.45, 0.32, 0.28
Tasks: 248 total,   1 running, 247 sleeping,   0 stopped,   0 zombie
%Cpu(s):  3.2 us,  1.1 sy,  0.0 ni, 95.6 id,  0.1 wa,  0.0 hi,  0.0 si,  0.0 st
MiB Mem :  11998.2 total,   4567.8 free,   6123.4 used,   1307.0 buff/cache
MiB Swap:  24576.0 total,  24576.0 free,      0.0 used.   5874.8 avail Mem

  PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND
 1234 nuoxi     20   0  123456  45678  12345 S   5.2   2.1   0:15.23 astrbot
 5678 root      20   0  234567  12345   6789 S   0.3   0.1   0:00.56 dockerd
```

### `top` 交互快捷键
| 按键 | 功能 |
|------|------|
| `P` | 按 CPU 使用率排序（**最常用**） |
| `M` | 按内存使用率排序 |
| `N` | 按 PID 排序 |
| `T` | 按运行时间排序 |
| `k` | 终止进程（输入 PID） |
| `r` | 修改进程优先级（renice） |
| `q` | 退出 |
| `h` | 显示帮助 |

### `htop` — 增强版（推荐）
```bash
htop
```

### `top` vs `htop` 对比
| 特性 | `top` | `htop` |
|------|-------|--------|
| 界面 | 单色，简单 | 彩色，美观 |
| 操作 | 键盘快捷键 | 支持鼠标点击 + 键盘 |
| 树形视图 | ❌ | ✅ 按 `F5` 切换进程树 |
| 搜索 | ❌ | ✅ 按 `F3` 搜索进程 |
| 进程筛选 | ❌ | ✅ 按 `F4` 过滤 |
| 横向滚动 | ❌ | ✅ 水平滚动查看完整命令 |
| 安装 | 默认已安装 | 需要安装 `sudo pacman -S htop` |

### 实际应用
```bash
# 查看 AstrBot 的资源占用
htop
# 按 F4，输入 astrbot，只显示相关进程

# 查看内存占用最高的进程
htop
# 按 F6，选择 MEM%，排序
```

## 4. `kill PID` — 终止进程

### 命令格式
```bash
kill PID              # 正常终止（发送 SIGTERM）
kill -9 PID           # 强制终止（发送 SIGKILL）
kill -15 PID          # 同 kill（SIGTERM）
kill -2 PID           # 中断信号（相当于 Ctrl+C）
```

### 信号类型（常用）
| 信号 | 数字 | 含义 | 行为 |
|------|------|------|------|
| `SIGTERM` | 15 | **终止**（默认） | 礼貌地请求进程退出，可被捕获 |
| `SIGKILL` | 9 | **强制终止** | 立即终止，无法被捕获或忽略 |
| `SIGINT` | 2 | 中断 | 相当于按 `Ctrl+C` |
| `SIGHUP` | 1 | 挂断 | 终端关闭时发送，常用于重载配置 |
| `SIGSTOP` | 19 | 暂停 | 暂停进程，不被捕获 |
| `SIGCONT` | 18 | 继续 | 恢复被暂停的进程 |
| `SIGUSR1` | 10 | 用户自定义1 | 一般用于触发特定操作 |

### 实际应用
```bash
# 正常终止 AstrBot
kill 1234

# 强制终止 AstrBot（如果正常 kill 无效）
kill -9 1234

# 重载 Nginx 配置（不中断服务）
kill -HUP $(cat /run/nginx.pid)

# 查看可用信号列表
kill -l
```

### ⚠️ `kill -9` 的使用原则
| 原则 | 说明 |
|------|------|
| ✅ **最后手段** | 当 `kill` 无法终止时才使用 |
| ✅ **明确知道后果** | 可能导致文件损坏或状态丢失 |
| ❌ **不要滥用** | 应优先使用 `kill -15` 让程序优雅退出 |
| ❌ **不要杀系统进程** | 可能导致系统不稳定 |

## 5. `pkill` — 按名称终止进程

### 命令格式
```bash
pkill 进程名           # 按名称终止
pkill -9 进程名        # 强制终止
pkill -f "完整命令"    # 匹配完整命令行
```

### 实际应用
```bash
# 终止所有 astrbot 进程
pkill astrbot

# 强制终止所有 python 脚本（慎用）
pkill -9 python

# 终止特定用户的所有进程
pkill -u nuoxi astrbot

# 匹配完整命令行（包含参数）
pkill -f "python.*astrbot"
```

### `pkill` vs `kill` 对比
| 特性 | `kill` | `pkill` |
|------|--------|---------|
| 目标 | 需要 PID | 支持进程名 |
| 精确度 | 高（精确 PID） | 中（可能误杀同名进程） |
| 适用场景 | 已知 PID | 只知道进程名称 |

## 6. `pgrep` — 按名称查找 PID

### 命令格式
```bash
pgrep 进程名
pgrep -f "完整命令"
pgrep -u 用户名 进程名
```

### 实际应用
```bash
# 查找 AstrBot 的 PID
pgrep astrbot
# 输出：1234

# 查找所有 python 进程的 PID
pgrep python

# 查找并返回完整进程名
pgrep -a astrbot
# 输出：1234 /usr/bin/python3 /usr/bin/astrbot

# 统计进程数量
pgrep -c astrbot
```

### 组合使用
```bash
# 终止 AstrBot（先查找再 kill）
kill $(pgrep astrbot)

# 查看 AstrBot 详细信息
ps -p $(pgrep astrbot) -o pid,cmd
```

## 7. `nice` / `renice` — 调整进程优先级

### 优先级范围
| 范围 | 含义 | 说明 |
|------|------|------|
| `-20` ~ `19` | Nice 值 | 数值越低优先级越高 |
| 默认值 | `0` | 普通进程 |
| 低优先级 | `+19` | 最不紧急的任务 |
| 高优先级 | `-20` | 最紧急的任务（需要 root 权限） |

### `nice` — 启动时设置优先级
```bash
nice -n 10 命令      # 以较低优先级运行
nice -n -5 命令      # 以较高优先级运行（需要 sudo）
```

### `renice` — 修改已有进程优先级
```bash
renice 10 -p PID     # 降低进程优先级
renice -5 -p PID     # 提高进程优先级（需要 sudo）
renice 10 -u nuoxi   # 调整用户所有进程的优先级
```

### 实际应用
```bash
# 以低优先级运行备份脚本（不影响其他任务）
nice -n 19 ./backup.sh

# 降低 AstrBot 的优先级（如果它占用了过多资源）
sudo renice 10 -p $(pgrep astrbot)

# 查看进程的优先级
ps -l -p $(pgrep astrbot)
```

## 8. `nohup` — 后台运行，忽略挂断信号

### 命令格式
```bash
nohup 命令 &          # 后台运行，忽略 SIGHUP
nohup 命令 > output.log 2>&1 &  # 重定向输出到文件
```

### 使用场景
当你关闭终端时，终端会向所有子进程发送 `SIGHUP` 信号，导致进程退出。`nohup` 让进程忽略这个信号。

### 实际应用
```bash
# 后台运行 AstrBot 并保存日志
nohup astrbot run > astrbot.log 2>&1 &

# 运行 Python 脚本，即使关闭终端也继续运行
nohup python3 long_task.py &

# 查看后台运行的 nohup 进程
jobs
ps aux | grep "nohup"
```

## 9. `jobs` / `fg` / `bg` — 作业控制

### 命令格式
```bash
jobs              # 查看当前终端后台任务
fg %编号          # 将后台任务调至前台
bg %编号          # 将挂起任务转为后台运行
```

### 使用场景
| 操作 | 命令 | 说明 |
|------|------|------|
| 暂停前台任务 | `Ctrl+Z` | 将前台任务挂起到后台（暂停） |
| 后台继续运行 | `bg %1` | 让挂起的任务在后台继续执行 |
| 调回前台 | `fg %1` | 将后台任务调回前台 |
| 查看后台任务 | `jobs` | 列出所有后台任务 |

### 实际应用
```bash
# 启动一个任务
python3 long_task.py
# 按 Ctrl+Z 暂停
[1]+  Stopped                 python3 long_task.py

# 查看后台任务
jobs
[1]+  Stopped                 python3 long_task.py

# 在后台继续运行
bg %1
[1]+ python3 long_task.py &

# 调回前台
fg %1
python3 long_task.py
```

## 💡 与 AstrBot 相关的实战组合

```bash
# 1. 查看 AstrBot 是否在运行
ps aux | grep astrbot | grep -v grep

# 2. 查看 AstrBot 的资源占用
htop  # 按 F4 搜索 astrbot

# 3. 重启 AstrBot（通过 systemd）
systemctl --user restart astrbot

# 4. 查看 AstrBot 的 PID
pgrep -a astrbot

# 5. 强制终止 AstrBot（如果卡住）
pkill -9 astrbot
# 然后重新启动
astrbot run

# 6. 后台运行 AstrBot 并保存日志
nohup astrbot run > ~/astrbot.log 2>&1 &
```
