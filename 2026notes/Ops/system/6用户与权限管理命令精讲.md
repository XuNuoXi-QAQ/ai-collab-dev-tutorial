# 用户与权限管理命令精讲

用户与权限是 Linux 安全模型的核心。这组命令帮助你管理系统的使用者、用户组，以及文件和目录的访问权限。

---

## 一、核心概念

### 用户（User）
- 每个用户有唯一的 **UID**（用户 ID）
- 普通用户 UID 通常 ≥ 1000
- root 用户的 UID 为 0

### 组（Group）
- 每个用户至少属于一个**主组**（primary group）
- 可以加入多个**附加组**（supplementary groups）
- 组的权限可被组内所有成员继承

### 文件权限三要素
| 元素 | 含义 | 示例 |
|------|------|------|
| **所有者**（Owner） | 文件的创建者 | `nuoxi` |
| **所属组**（Group） | 文件关联的组 | `users` |
| **其他用户**（Others） | 非所有者、非组内成员 | `others` |

### 权限类型
| 权限 | 含义 | 数字表示 |
|------|------|----------|
| `r` | 读（read） | 4 |
| `w` | 写（write） | 2 |
| `x` | 执行（execute） | 1 |
| `-` | 无权限 | 0 |

## 2. `whoami` — 查看当前用户

### 命令格式
```bash
whoami
```

### 输出示例
```
nuoxi
```

### 实际应用
```bash
# 在脚本中获取当前用户名
CURRENT_USER=$(whoami)
echo "当前用户是 $CURRENT_USER"

# 对比：whoami 与 who am i 的区别
whoami      # 只显示用户名
who am i    # 显示用户名、登录终端、登录时间
```

## 3. `id` — 查看用户 UID/GID

### 命令格式
```bash
id              # 查看当前用户
id 用户名        # 查看指定用户
```

### 输出示例
```bash
id
```
```
uid=1000(nuoxi) gid=1000(nuoxi) groups=1000(nuoxi),998(wheel),100(docker),44(video)
```

### 各部分含义
| 字段 | 含义 | 示例值 |
|------|------|--------|
| `uid` | 用户 ID | `1000(nuoxi)` |
| `gid` | 主组 ID | `1000(nuoxi)` |
| `groups` | 所有附加组 | `wheel`, `docker`, `video` |

### 常用参数
| 参数 | 用途 |
|------|------|
| `-u` | 只显示 UID |
| `-g` | 只显示主组 GID |
| `-G` | 显示所有组 GID |
| `-n` | 显示名称而非数字 |

### 实际应用
```bash
# 查看当前用户 UID
id -u
# 输出：1000

# 查看当前用户主组
id -gn
# 输出：nuoxi

# 查看 nuoxi 是否在 docker 组中
id nuoxi | grep docker
```

## 4. `sudo useradd -m 用户名` — 创建新用户

### 命令格式
```bash
sudo useradd -m 用户名        # 创建用户并创建主目录
sudo useradd -m -G 组 用户名  # 创建用户并加入附加组
sudo useradd -m -s /bin/zsh 用户名  # 指定 shell
```

### 参数说明
| 参数 | 含义 |
|------|------|
| `-m` | 创建用户主目录（`/home/用户名`） |
| `-G` | 指定附加组（逗号分隔） |
| `-s` | 指定登录 shell |
| `-u` | 指定 UID |
| `-c` | 添加注释（如全名） |

### 实际应用
```bash
# 创建新用户
sudo useradd -m -s /bin/bash newuser

# 创建用户并加入 docker 组
sudo useradd -m -G docker newuser

# 创建用户并指定 UID
sudo useradd -m -u 1001 newuser

# 创建用户并设置全名
sudo useradd -m -c "John Doe" johndoe
```

### ⚠️ 注意：`useradd` 与 `adduser` 的区别
| 命令 | 来源 | 行为 |
|------|------|------|
| `useradd` | 系统命令 | 较底层，默认不创建主目录（需 `-m`） |
| `adduser` | 交互式脚本（Debian 系） | 更友好，会自动创建主目录并设置密码 |

在 Manjaro/Arch 中，推荐使用 `useradd -m`。

## 5. `sudo passwd 用户名` — 设置用户密码

### 命令格式
```bash
sudo passwd 用户名      # 设置/修改密码
passwd                  # 修改当前用户密码
```

### 执行过程
```
$ sudo passwd newuser
New password: （输入密码，不显示）
Retype new password: （再次输入）
passwd: password updated successfully
```

### 实际应用
```bash
# 为新用户设置密码
sudo passwd newuser

# 修改当前用户自己的密码
passwd

# 强制用户下次登录时修改密码
sudo passwd -e 用户名
# 或
sudo chage -d 0 用户名
```

### 密码安全建议
| 原则 | 说明 |
|------|------|
| 长度 | ≥ 8 位 |
| 复杂度 | 包含大小写字母、数字、特殊字符 |
| 不与用户名相同 | 避免使用 `password`、`123456` 等 |
| 定期更换 | 建议每 90 天更换一次 |

## 6. `sudo usermod -aG 组 用户` — 将用户加入组

### 命令格式
```bash
sudo usermod -aG 组名 用户名    # 追加到附加组
sudo usermod -aG 组1,组2 用户名 # 加入多个组
sudo usermod -G 组名 用户名     # 设置主组（覆盖）
```

### 参数说明
| 参数 | 含义 |
|------|------|
| `-a` | **追加**（append）而不是覆盖 |
| `-G` | 指定附加组列表 |
| `-g` | 修改主组（primary group） |

### ⚠️ 重点：`-a` 参数很重要
```bash
# ❌ 错误用法：会覆盖所有组，将用户从其他组移除
sudo usermod -G docker nuoxi

# ✅ 正确用法：追加到 docker 组
sudo usermod -aG docker nuoxi
```

### 实际应用（你之前用过）
```bash
# 将 nuoxi 加入 docker 组
sudo usermod -aG docker nuoxi

# 将 nuoxi 加入多个组
sudo usermod -aG docker,video,input nuoxi

# 查看用户所属组
groups nuoxi
# 输出：nuoxi wheel docker video
```

### 组权限生效
加入组后需要**重新登录**或**启动新 Shell** 才能使组权限生效：
```bash
# 方法一：重新登录
exit
# 然后重新登录

# 方法二：启动新 Shell
newgrp docker
# 或
su - nuoxi

# 方法三：手动刷新组（较少用）
exec su -l $USER
```

## 7. `sudo userdel -r 用户名` — 删除用户

### 命令格式
```bash
sudo userdel -r 用户名    # 删除用户及主目录
sudo userdel 用户名       # 仅删除用户（保留主目录）
```

### 参数说明
| 参数 | 含义 |
|------|------|
| `-r` | 删除用户主目录和邮件池 |
| `-f` | 强制删除（即使用户已登录） |

### 实际应用
```bash
# 删除用户及主目录（推荐）
sudo userdel -r olduser

# 仅删除用户，保留文件
sudo userdel olduser

# 强制删除（用户可能登录）
sudo userdel -rf olduser
```

### ⚠️ 删除前的检查
```bash
# 查看用户是否登录
who | grep olduser

# 查看用户是否有运行中的进程
ps -u olduser

# 先终止用户进程
sudo pkill -u olduser
# 再删除用户
sudo userdel -r olduser
```

## 8. `groups 用户名` — 查看用户所属组

### 命令格式
```bash
groups              # 查看当前用户所属组
groups 用户名       # 查看指定用户所属组
```

### 输出示例
```bash
groups nuoxi
```
```
nuoxi : nuoxi wheel docker video
```

### 实际应用
```bash
# 检查是否在 docker 组中
groups nuoxi | grep docker

# 检查是否在 wheel 组中（sudo 权限）
groups nuoxi | grep wheel

# 查看所有组信息（包括组 ID）
getent group
```

## 9. `chown 用户:组 文件` — 修改文件所有者

### 命令格式
```bash
sudo chown 用户 文件           # 修改所有者
sudo chown :组 文件            # 修改所属组
sudo chown 用户:组 文件        # 同时修改所有者与组
sudo chown -R 用户:组 目录     # 递归修改目录及子文件
```

### 实际应用
```bash
# 修改文件所有者
sudo chown nuoxi file.txt

# 修改文件所属组
sudo chown :users file.txt

# 同时修改所有者和组
sudo chown nuoxi:users file.txt

# 递归修改目录（你之前用过）
sudo chown -R nuoxi:nuoxi /home/nuoxi/Project
```

### 常见使用场景
| 场景 | 命令 |
|------|------|
| 修复个人文件权限 | `sudo chown -R nuoxi:nuoxi ~/*` |
| 共享目录给组 | `sudo chown -R :docker /opt/docker-data` |
| 恢复系统文件 | `sudo chown root:root /etc/passwd` |

## 10. `chmod` — 修改文件权限

### 命令格式（数字模式）
```bash
chmod 权限 文件
```
权限由三位数字组成：**所有者权限 + 组权限 + 其他用户权限**。

| 数字 | 权限 |
|------|------|
| 7 | 读 + 写 + 执行（4+2+1） |
| 6 | 读 + 写（4+2） |
| 5 | 读 + 执行（4+1） |
| 4 | 只读 |
| 3 | 写 + 执行（2+1） |
| 2 | 只写 |
| 1 | 只执行 |
| 0 | 无权限 |

### 常用权限组合
| 权限数字 | 含义 | 适用场景 |
|----------|------|----------|
| `755` | 所有者 7，组 5，其他 5 | 可执行文件/目录 |
| `644` | 所有者 6，组 4，其他 4 | 普通文件 |
| `777` | 所有人可读写执行 | ⚠️ 极度危险，避免使用 |
| `600` | 所有者 6，组 0，其他 0 | 私密文件（如 SSH 密钥） |

### 命令格式（符号模式）
```bash
chmod u+x 文件        # 为所有者添加执行权限
chmod g+w 文件        # 为组添加写权限
chmod o-r 文件        # 移除其他用户的读权限
chmod a+x 文件        # 为所有人添加执行权限
```

| 符号 | 含义 |
|------|------|
| `u` | 所有者（user） |
| `g` | 组（group） |
| `o` | 其他用户（others） |
| `a` | 所有人（all） |
| `+` | 添加权限 |
| `-` | 移除权限 |
| `=` | 设置权限（覆盖） |

### 实际应用
```bash
# 设置脚本可执行（你常用的命令）
chmod +x script.sh
# 等同于 chmod 755 script.sh

# 设置文件仅所有者可读写
chmod 600 private.txt

# 递归设置目录权限
chmod -R 755 ~/bin/

# 为组添加写权限
chmod g+w shared_file.txt

# 移除其他用户的读权限
chmod o-r report.pdf
```

## 11. `sudo -i` — 切换到 root 用户

### 命令格式
```bash
sudo -i             # 切换到 root 用户（交互式）
```

### 与 `sudo -s` 的区别
| 命令 | 行为 |
|------|------|
| `sudo -i` | 切换到 root，并加载 root 的配置文件（`~/.bashrc`） |
| `sudo -s` | 切换到 root，但保留当前用户的环境变量 |
| `sudo su -` | 传统方式，加载 root 环境 |

### 实际应用
```bash
# 进入 root 交互式 Shell
sudo -i
# 现在你是 root，可以执行任何命令
exit
# 回到普通用户

# 以 root 执行单条命令（不切换）
sudo command

# 以 root 执行命令，保留当前用户环境
sudo -E command
```

## 12. `sudo -E` — 以当前用户环境执行 sudo

### 命令格式
```bash
sudo -E 命令       # 保留当前用户环境变量执行命令
```

### 实际应用
```bash
# 安装 pip 包时保留当前用户代理环境
sudo -E pip install some-package

# 使用当前用户 PATH 执行命令
sudo -E which python
```

### 为什么需要 `-E`？
默认情况下，`sudo` 会重置环境变量（如 `PATH`、`HTTP_PROXY`），使用 `-E` 可以保留这些变量。

## 13. `su - 用户名` — 切换用户

### 命令格式
```bash
su - 用户名        # 切换到指定用户（加载环境）
su 用户名          # 切换到指定用户（不加载环境）
su -               # 切换到 root（需 root 密码）
```

### `su -` vs `su` 的区别
| 命令 | 行为 |
|------|------|
| `su - nuoxi` | 切换到 nuoxi，加载其环境变量 |
| `su nuoxi` | 切换到 nuoxi，保留当前环境 |

### 实际应用
```bash
# 切换到 nuoxi 用户
su - nuoxi

# 以其他用户身份执行命令
su - nuoxi -c "ls -la ~/documents"
```

## 14. `visudo` — 编辑 sudoers 文件

### 命令格式
```bash
sudo visudo        # 编辑 /etc/sudoers
```

### ⚠️ 重要说明
- `visudo` 会检查语法错误，防止 sudoers 损坏
- **不要**直接编辑 `/etc/sudoers`，使用 `visudo`
- `visudo` 默认使用 `nano` 作为编辑器

### 常见配置
```bash
# 让用户无需密码执行 sudo
nuoxi ALL=(ALL) NOPASSWD: ALL

# 让用户只能执行特定命令
nuoxi ALL=(ALL) /usr/bin/systemctl restart docker

# 让组内的用户拥有 sudo 权限
%wheel ALL=(ALL) ALL
```

### 实际应用
```bash
# 打开 visudo 并添加配置
sudo visudo

# 在文件末尾添加：
nuoxi ALL=(ALL) ALL

# 保存退出（nano: Ctrl+O, Ctrl+X）
```

## 15. `last` — 查看最近登录记录

### 命令格式
```bash
last              # 查看所有登录记录
last -n 10        # 查看最近 10 条
last reboot       # 查看重启记录
```

### 输出示例
```
nuoxi   pts/0        192.168.1.100   Mon Jun 22 14:30   still logged in
nuoxi   pts/1        192.168.1.100   Mon Jun 22 10:15 - 12:30  (02:15)
reboot   system boot  6.18.33-1-MANJ Mon Jun 22 09:00 - 14:50  (05:50)
```

### 实际应用
```bash
# 检查是否有异常登录
last | head -20

# 查看本月登录记录
last | grep Jun

# 查看特定用户的登录记录
last nuoxi
```

## 16. `who` — 查看当前登录用户

### 命令格式
```bash
who               # 查看当前登录用户
who -a            # 显示详细信息
```

### 输出示例
```
nuoxi   pts/0        2026-06-22 14:30 (192.168.1.100)
root    pts/1        2026-06-22 14:35 (192.168.1.1)
```

### 实际应用
```bash
# 查看当前登录用户
who

# 查看自己
who am i

# 查看登录用户数

```

## 💡 你的实战场景

### 场景1：将用户加入 Docker 组（你之前做过）
```bash
# 将 nuoxi 加入 docker 组
sudo usermod -aG docker nuoxi

# 验证
groups nuoxi

# 重新登录后生效
exit
# 重新登录
```

### 场景2：修复文件权限问题
```bash
# 场景：一个文件无法编辑
ls -la file.txt
# -rw-r--r-- 1 root root file.txt

# 修改所有者
sudo chown nuoxi:nuoxi file.txt

# 修改权限
chmod 644 file.txt
```

### 场景3：设置脚本可执行（你经常用）
```bash
# 给脚本添加执行权限
chmod +x script.sh

# 查看权限
ls -la script.sh
# -rwxr-xr-x 1 nuoxi nuoxi script.sh
```

### 场景4：安全配置 sudo 免密（可选）
```bash
# 编辑 sudoers
sudo visudo

# 添加一行：
nuoxi ALL=(ALL) NOPASSWD: ALL

# 保存后即可在不输入密码的情况下执行 sudo
```

---

## ⚠️ 安全提醒

1. **`chmod 777` 极不安全**：避免给文件设置 777 权限。
2. **`sudo` 慎用**：只在必要时使用，避免以 root 身份运行普通程序。
3. **定期检查登录记录**：使用 `last` 和 `who` 监控异常登录。
4. **密码安全**：使用强密码，定期更换。
5. **`visudo` 优先**：编辑 sudoers 时必须使用 `visudo`，不要直接编辑文件。
