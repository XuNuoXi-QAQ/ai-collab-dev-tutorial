# OpenCode 使用指南

> OpenCode 是一个开源的 AI 编程助手，支持终端、桌面和 IDE 多种形态，让 AI 辅助编程更高效、更安全。

---

## 🚀 什么是 OpenCode

### 简介

**OpenCode** 是一个开源的 AI 编程助手，它不仅仅是一个代码补全工具，而是一个完整的 AI 编程伙伴。它能够理解你的项目、编写代码、调试问题、重构优化，甚至帮你管理 Git 版本。

### 核心数据

- ⭐ **GitHub 星标**：50,000+
- 👥 **月活用户**：65万+
- 🔌 **支持模型**：75+ 大模型
- 🧩 **插件数量**：40+ 插件
- 💻 **多平台**：终端 TUI、桌面应用、IDE 扩展

### 核心特点

| 特点 | 说明 |
|------|------|
| **开源免费** | 完全开源，无订阅费，无使用限制 |
| **多模型支持** | 支持 75+ 大模型，可自由切换 |
| **本地运行** | 支持 Ollama 本地模型，可完全离线 |
| **隐私安全** | 会话数据、代码、配置都存储在本地 |
| **LSP 支持** | 内置语言服务器协议，智能代码分析 |
| **多会话并行** | 同一项目上并行多个助手会话 |
| **插件化架构** | 40+ 插件，按需扩展功能 |
| **Git 集成** | 内置版本控制，协作更便捷 |

## 📦 安装与配置

### 系统要求

- **操作系统**：Windows / macOS / Linux
- **Node.js**：建议 18+ 版本
- **Git**：版本控制支持
- **可选**：Ollama（本地模型）

### 安装方法

#### 方法一：一键脚本安装（推荐）

```bash
curl -fsSL https://opencode.ai/install | bash
```

#### 方法二：使用 npm 安装

```bash
npm install -g opencode-ai
```

#### 方法三：使用 Homebrew（macOS）

```bash
brew tap opencode-ai/opencode
brew install opencode
```

#### 方法四：桌面应用

1. 访问官方下载页
2. 选择对应平台的安装包
3. 按照向导安装

#### 方法五：IDE 扩展

- VS Code 扩展
- JetBrains 插件
- Neovim 插件

### 初始配置

#### 1. 首次启动

```bash
opencode
```

首次启动会引导你完成配置：

1. 选择默认模型提供商
2. 配置 API 密钥（或使用免费模型）
3. 设置主题和快捷键
4. 初始化项目配置

#### 2. 配置模型提供商

OpenCode 支持多种模型提供商：

| 提供商 | 配置方式 | 特点 |
|--------|----------|------|
| **OpenAI** | API Key | 能力强，速度快 |
| **Anthropic** | API Key | 长文本，代码好 |
| **Google** | API Key | 多模态强 |
| **Ollama** | 本地端点 | 免费，离线，隐私 |
| **DeepSeek** | API Key | 国产，代码好 |
| **硅基流动** | API Key | 国内，多模型 |

**配置 Ollama 本地模型**：

```bash
# 1. 先安装 Ollama
# 2. 下载模型
ollama pull codellama

# 3. 在 OpenCode 中配置
# 设置 → 模型提供商 → 添加 Ollama
# 端点：http://localhost:11434
```

### 常用配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `defaultModel` | 默认使用的模型 | 依提供商而定 |
| `theme` | 界面主题 | dark |
| `keybindings` | 快捷键方案 | vscode |
| `formatOnSave` | 保存时自动格式化 | true |
| `gitIntegration` | 是否启用 Git 集成 | true |
| `autoComplete` | 是否启用自动补全 | true |

## 🎯 核心功能详解

### 1. 代码编辑功能

#### 智能代码补全

- 实时代码补全建议
- 支持多种编程语言
- 基于上下文的智能建议
- Tab 键接受补全

#### 代码生成

**方式一：自然语言描述**
```
// 请写一个函数，验证邮箱格式是否正确
```

**方式二：选中代码 + 指令**
```
选中代码 → 右键 → AI 编辑 → 重构这个函数
```

**方式三：快捷键触发**
- `Ctrl+K`：打开 AI 命令面板
- `Ctrl+I`：行内 AI 编辑

#### 代码解释

选中代码后，可以让 AI 解释：

```
请解释这段代码的功能和逻辑
```

适用场景：
- 阅读别人的代码
- 学习新的技术
- 理解复杂的算法

### 2. AI 辅助编程功能

#### 代码重构

常用的重构操作：
- 重命名变量/函数
- 提取函数/类
- 优化代码结构
- 改进性能
- 增加类型注解

**示例**：
```
请重构这个函数：
1. 拆分成更小的函数
2. 增加类型注解
3. 添加错误处理
4. 优化性能
```

#### 代码审查

让 AI 帮你审查代码：

```
请审查这段代码，从以下角度评估：
1. 功能正确性
2. 代码质量
3. 安全性
4. 性能
5. 可维护性

指出问题并给出改进建议。
```

#### Bug 修复

```
这段代码有 bug，现象是...
请帮我定位问题并修复。
```

#### 测试生成

```
请为这个函数编写单元测试：
- 使用 Jest 测试框架
- 覆盖正常情况和边界情况
- 包含错误处理测试
```

### 3. 调试功能

#### 错误分析

当代码报错时，直接把错误信息发给 AI：

```
报错信息：
TypeError: Cannot read property 'name' of undefined
    at getUserInfo (user.js:42:15)

请帮我分析可能的原因和修复方案。
```

#### 调试建议

```
这个功能不工作，我该怎么调试？
请给我一个调试步骤清单。
```

### 4. 版本控制集成（Git）

#### 智能提交信息

```
请根据我的代码变更，生成一个符合 Conventional Commits 规范的提交信息。
```

#### 变更总结

```
请总结一下这个 PR 的主要变更：
1. 新增了什么功能
2. 修复了什么问题
3. 有哪些破坏性变更
```

#### Code Review 辅助

```
请帮我 review 这个 PR 的代码变更：
- 有没有明显的 bug
- 代码风格是否一致
- 有没有可以优化的地方
```

### 5. 终端集成

OpenCode 内置了终端，可以：
- 直接运行命令
- AI 帮你写命令
- 解释命令的作用
- 调试命令错误

**示例**：
```
我想查看当前目录下大于 100MB 的文件，该用什么命令？
```

### 6. 多会话并行

你可以在同一个项目中打开多个 AI 会话：
- 会话 A：写前端代码
- 会话 B：写后端 API
- 会话 C：调试问题

每个会话独立上下文，互不干扰。

### 7. 分享功能

可以将会话分享给别人：
- 生成分享链接
- 导出会话记录
- 导入会话继续

适用场景：
- 团队协作
- 教学演示
- Bug 复现

### 提高效率的技巧

OpenCode 支持很多斜杠命令：

| 命令 | 功能 |
|------|------|
| `/help` | 查看帮助 |

在对话中引用文件：

```
请帮我优化 @src/utils/helpers.js 中的 debounce 函数
```

或者直接拖拽文件到对话框。

让 AI 同时处理多个文件：

```

```

你可以创建自定义的 AI 命令，快速执行常用操作。

## 💡 最佳实践

### 1. 清晰描述需求

**不好的**：
```
帮我优化这段代码
```

**好的**：
```
请优化这段 React 组件代码，目标：
1. 提升渲染性能（减少不必要的重渲染）
2. 改进代码结构（提取子组件）
3. 增加 TypeScript 类型注解
4. 保持功能不变

代码：
...
```

### 2. 提供上下文

- 告诉 AI 项目的技术栈
- 提供相关的代码文件
- 说明项目的代码规范
- 给出参考示例

### 3. 分步骤进行

不要一次让 AI 做太多事情：

```
第一步：先设计数据结构
第二步：再实现核心逻辑
第三步：然后加 UI 样式
第四步：最后优化和测试
```

### 4. 迭代优化

第一版通常不是最好的，持续迭代：

```
第一版不错，但是有几个问题：
1. xxx 地方可以改进
2. 需要增加 xxx 功能
3. 性能还可以优化

请再调整一下。
```

### 5. 审查代码

AI 生成的代码一定要自己审查：
- 逻辑是否正确？
- 有没有安全问题？
- 性能怎么样？
- 符合项目规范吗？

### 6. 善用本地模型

对于敏感代码，使用本地模型：
- 安装 Ollama
- 下载代码模型（如 codellama、deepseek-coder）
- 在 OpenCode 中配置使用
- 完全离线，隐私安全

## 🤝 与 AI Agent 结合使用

### OpenCode + 多 Agent 架构

OpenCode 可以作为多 Agent 架构中的**开发型 SubAgent**：

```
MainAgent（用户交互）
    ↓
OrganizeAgent（任务调度）
    ↓
┌─────────────┬─────────────┐
↓             ↓             ↓
研究型      创作型      开发型
Agent       Agent       Agent
                        ↳ OpenCode
```

### 协作工作流

#### 场景一：新功能开发

```
1. 用户提出需求
   ↓
2. OrganizeAgent 分析任务
   ↓
3. 研究型 Agent 做技术调研
   ↓
4. 设计型 Agent 出设计方案
   ↓
5. OpenCode（开发型 Agent）实现代码
   ↓
6. 审校型 Agent 做 Code Review
   ↓
7. 测试型 Agent 写测试
   ↓
8. 整合交付
```

#### 场景二：Bug 修复

```
1. 用户报告 bug
   ↓
2. OpenCode 分析错误信息
   ↓
3. 定位问题原因
   ↓
4. 修复代码
   ↓
5. 验证修复
   ↓
6. 生成提交信息
```

### 优势

1. **专注代码**：OpenCode 专注编程场景，更专业
2. **上下文丰富**：能读取整个项目，理解更深入
3. **工具齐全**：内置 Git、终端、调试等工具
4. **效率更高**：比通用 Agent 写代码更快更好

## ❓ 常见问题与解决方案

### 1. 安装问题

**Q: 安装失败怎么办？**

A:
- 检查 Node.js 版本（需要 18+）
- 尝试使用管理员权限
- 检查网络连接
- 换用其他安装方式

### 2. 模型配置问题

**Q: 如何添加自定义模型？**

A:
1. 打开设置 → 模型提供商
2. 选择 "添加自定义提供商"
3. 填写 API 端点和密钥
4. 选择对应的模型格式（OpenAI 兼容等）

**Q: Ollama 连接失败？**

A:
- 确认 Ollama 正在运行：`ollama serve`
- 检查端口是否正确（默认 11434）
- 确认模型已经下载：`ollama list`

### 3. 使用问题

**Q: AI 回答太慢怎么办？**

A:
- 换用更快的模型
- 减少上下文长度
- 检查网络连接
- 使用本地模型（如果机器性能好）

**Q: 代码补全不出现？**

A:
- 检查自动补全是否开启
- 确认文件类型被支持
- 试试手动触发（Ctrl+Space）
- 检查 LSP 是否正常运行

### 4. 性能问题

**Q: OpenCode 占用资源太多？**

A:
- 关闭不需要的插件
- 减少同时打开的会话数
- 调整 LSP 配置
- 使用轻量模式
