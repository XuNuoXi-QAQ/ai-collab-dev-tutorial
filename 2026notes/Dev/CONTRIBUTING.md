# 贡献指南

首先，感谢你花时间为这个项目做贡献！🎉

我们欢迎所有形式的贡献，无论是：

- 🐛 报告 Bug
- 💡 提出新功能建议
- 📝 改进文档
- 💻 提交代码
- 🔍 参与代码审查

请阅读以下内容，了解如何参与贡献。

---

## 行为准则

参与本项目的所有人都需要遵守我们的行为准则：

- 🤝 尊重他人，友善沟通
- 🌟 对事不对人，专注于问题本身
- 🙏 虚心接受不同意见
- 💡 乐于分享和帮助他人

---

## 如何开始

如果你是第一次贡献，可以从以下几个方面入手：

### 1. 报告 Bug

如果你发现了 Bug，请通过 Issue 告诉我们：

1. 先搜索一下有没有人已经报告过同样的问题
2. 如果没有，创建一个新的 Issue
3. 按照 Bug 模板填写详细信息
4. 提供复现步骤和期望行为

### 2. 提出功能建议

有好的想法？欢迎提出来！

1. 先搜索一下有没有类似的建议
2. 创建一个新的 Issue，标签选 `enhancement`
3. 详细描述你的想法和使用场景
4. 说明为什么这个功能对大家有用

### 3. 改进文档

文档永远可以更好！

- 发现错别字或表述不清？
- 觉得某个部分可以更详细？
- 有更好的示例？

都欢迎提交 PR 来改进！

### 4. 贡献代码

想直接写代码？太棒了！

- 可以从标有 `good first issue` 的 Issue 开始
- 也可以认领你感兴趣的 Issue
- 不确定做什么？可以在 Issue 里问

## 开发流程

### 前置条件

- Git
- 一个 GitHub 账号
- 基本的 Markdown 知识（本项目主要是文档）

### 工作流

我们使用 GitHub Flow 工作流：

```
1. Fork 本仓库
   ↓
2. 创建你的功能分支
   ↓
3. 进行修改
   ↓
4. 提交你的更改
   ↓
5. 推送到你的 Fork
   ↓
6. 创建 Pull Request
   ↓
7. 等待 Code Review
   ↓
8. 根据反馈修改（如果需要）
   ↓
9. 合并！🎉
```

### 详细步骤

**第 1 步：Fork 并克隆**

```bash
# 在 GitHub 上点击 Fork 按钮

# 克隆你的 Fork
git clone https://github.com/你的用户名/ai-collab-dev-tutorial.git
cd ai-collab-dev-tutorial

# 添加上游仓库
git remote add upstream https://github.com/XuNuoXi-QAQ/ai-collab-dev-tutorial.git
```

**第 2 步：同步最新代码**

```bash
# 切换到主分支
git checkout main

# 拉取上游最新代码
git fetch upstream
git merge upstream/main
```

**第 3 步：创建功能分支**

```bash
# 创建新分支，从 main 分支切出
git checkout -b feature/你的功能名称

# 或者如果是修复 Bug
git checkout -b fix/问题描述
```

**分支命名规范：**

- `feature/xxx`：新功能
- `fix/xxx`：Bug 修复
- `docs/xxx`：文档改进
- `refactor/xxx`：代码重构

**第 4 步：进行修改**

- 按照项目的代码风格编写
- 保持代码简洁清晰
- 添加必要的注释
- 更新相关文档

**第 5 步：提交更改**

```bash
# 查看修改了哪些文件
git status

# 添加修改
git add .

# 提交
git commit -m "简短描述你的变更"
```

**提交信息规范：**

我们使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<scope>): <subject>
```

常用 type：

- `feat`：新功能
- `fix`：Bug 修复
- `docs`：文档
- `style`：格式调整
- `refactor`：重构
- `test`：测试
- `chore`：杂项

示例：
```
feat: 添加 AI 工具对比表格
fix: 修正第 3 章的错别字
docs: 更新贡献指南
```

**第 6 步：推送到你的 Fork**

```bash
git push origin feature/你的功能名称
```

**第 7 步：创建 Pull Request**

1. 打开你的 Fork 页面
2. 点击 "Compare & pull request"
3. 填写 PR 描述（使用 PR 模板）
4. 选择审查者
5. 点击 "Create pull request"

**第 8 步：Code Review**

- 我们会尽快审查你的 PR（通常 24 小时内）
- 审查者可能会提出一些修改建议
- 请根据反馈进行修改
- 修改后推送到同一个分支，PR 会自动更新

**第 9 步：合并！**

审查通过后，你的 PR 就会被合并到主分支！🎉

## Pull Request 要求

为了让你的 PR 能够快速通过，请确保：

### 内容要求

- [ ] 一个 PR 只做一件事
- [ ] 变更不要太大（建议不超过 400 行）
- [ ] 有清晰的 PR 描述
- [ ] 关联相关的 Issue

### 质量要求

- [ ] 内容准确无误
- [ ] 语言通顺，没有错别字
- [ ] 格式统一，符合项目规范
- [ ] 链接都能正常打开

---

## 代码审查

### 审查者会关注什么？

1. **正确性**：内容是否准确？
2. **完整性**：有没有遗漏？
3. **可读性**：是否清晰易懂？
4. **一致性**：是否符合项目整体风格？
5. **价值**：对项目和用户有帮助吗？

### 如何处理审查意见？

- ✅ **认真阅读**每条意见
- ✅ **有道理就改**，并回复 "已修改"
- ✅ **不同意可以讨论**，说明你的理由
- ✅ **有疑问就问**，不要不好意思
- ✅ **修改后重新请求审查**

## 常见问题

### Q: 我可以做什么贡献？

A: 任何形式的贡献都欢迎！即使只是改一个错别字也是很大的帮助。

### Q: 我是新手，会不会很困难？

A: 不会的！我们有 `good first issue` 标签的任务，特别适合新手。而且我们的社区非常友好，有问题随时问。

### Q: 贡献有什么要求吗？

A: 只要遵守我们的行为准则，认真对待就好。代码质量可以慢慢提升，参与的态度最重要。

### Q: 我的 PR 多久会被审查？

A: 我们尽量在 24 小时内响应。如果比较忙，可能会稍慢一点，请耐心等待。

### Q: 审查意见我不同意怎么办？

A: 可以在评论里讨论，说明你的理由。如果达不成一致，可以找项目负责人来仲裁。
