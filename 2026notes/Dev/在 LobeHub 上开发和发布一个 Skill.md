在 LobeHub 上开发和发布一个 Skill，核心是遵循其关于 **`SKILL.md` 文件**的规范，并通过**社区审核**来完成收录。

### 📝 开发规范：核心是 `SKILL.md`

每个 Skill 本质上是一个包含 `SKILL.md` 文件的独立包。它的核心是一份包含 **YAML 元数据** 和 **Markdown 正文** 的 `SKILL.md` 文件。

**1. 目录结构**
```
skill-name/
├── SKILL.md          # 必需，元数据+正文
└── 资源 (可选)
    ├── scripts/      # 可执行脚本 (Python/Bash等)
    ├── references/   # 按需加载的参考文档
    └── assets/       # 模板、图片等输出素材
```

**2. `SKILL.md` 元数据 (YAML Frontmatter)**
元数据位于文件顶部，由 `---` 包裹，是决定 Agent 何时调用此 Skill 的关键。

*   **`name` (必需)**: 技能标识符，只能包含**小写字母、数字和连字符(-)** 。
*   **`description` (必需)**: 最重要的字段，决定了技能的“命运”。
    *   **以主动动词开头**：“处理”、“生成”、“分析”。
    *   **包含触发短语**：“当需要...时使用”。
    *   **具体明确**：指明工具、语言或操作，避免空泛。
    *   **长度适中**：**50-200个字符**为佳。
*   **`license` (可选)**: 如 `Apache-2.0`。
*   **其他元数据 (可选)**: 包括 `metadata` 字段，可包含 `clawdbot.emoji`（表情符号）、`clawdbot.requires.anyBins`（依赖的CLI工具）等信息。

**3. `SKILL.md` 正文 (Markdown Body)**
元数据之后是 Markdown 格式的正文。

*   **结构清晰**：通常包含 `# 标题`、`## 摘要`、`## 何时使用`、`## 核心概念` 等章节。
*   **渐进式披露 (Progressive Disclosure)**：这是优化上下文窗口的关键。
    *   **第一级 (元数据)**: 始终在上下文中，用于匹配触发。
    *   **第二级 (正文)**: 技能被触发后才加载。
    *   **第三级 (资源)**: 按需加载，如脚本、参考资料等。
*   **保持精炼**：**正文建议控制在500行以内**。通用知识不必赘述，只提供模型不具备的专属知识。
*   **设置合适的自由度**：根据任务特性，给予AI文本指导（高自由）、带参数的伪代码（中自由）或具体脚本（低自由）。

### 🚀 如何被 LobeHub 收录？

提交 Skill 到 LobeHub 主要通过 GitHub 的 Pull Request (PR) 流程。

1.  **准备技能**：确保你的技能遵循上述所有规范，并通过本地验证。
2.  **Fork 仓库**：访问 LobeHub 的技能仓库（如 `lobehub/lobe-chat-plugins`）并 Fork。
3.  **添加技能**：将你的技能文件夹放入仓库的指定目录。
4.  **提交 Pull Request**：向官方仓库发起 PR，详细描述你的技能。
5.  **代码审查**：LobeHub 团队会审查你的 PR，可能包括对 `SKILL.md` 的规范性、内容质量等进行审核。
6.  **审核与合并**：通过审核后，你的 PR 会被合并，技能即成功上架。
7.  **发布与维护**：技能上线后，用户可安装使用。你可以通过更新 GitHub 仓库来迭代版本。

> **提示**：在 LobeHub 上，你也可以发布包含特定提示词、技能和知识库的“**助理**”。发布助理的前提是创建**社区个人档案**，流程类似，也有相应的内容质量指南。
