## 一、概述与核心价值

### 1.1 什么是 HAPI？

HAPI（Happy Coder 的本地优先开源替代方案）是一个**本地优先（Local-First）的 AI 编程远程控制框架**。它让你能通过手机、平板或任何能上网的设备，安全地远程操控运行在你电脑上的 AI 编程助手（如 Claude Code、OpenCode、Codex、Gemini CLI 等）。

简单说，HAPI 是一把"远程钥匙"——它不替代你现有的 AI 工具，而是充当一座**桥梁**，让内网 AI 突破物理位置的限制。

### 1.2 一句话定位

> **HAPI = 你的 AI 编程助手的"远程遥控器"**

### 1.3 HAPI vs Happy：关键区别

HAPI 是基于 Happy 二次开源的远程 Vibe Coding 工具。两者最大的区别在于数据存储方式：

| 对比项 | HAPI | Happy |
|--------|------|-------|
| **数据存储** | 全部保存在本地电脑，不上服务器 | 云端托管 |
| **适用场景** | 数据自主、自托管、最小设置 | 多用户协作、云端托管 |
| **国内访问** | 可自建中继，稳定可用 | App 经常卡顿、闪退 |
| **架构复杂度** | 简单轻量 | 复杂 |

### 1.4 技术栈与开源协议

- **CLI 部分**：MIT 许可证
- **Server 和 Web 部分**：LGPLv2
- **运行环境**：Bun（启动速度快，对现代 Web 标准支持好）
- **安装方式**：支持 npm/npx、Homebrew、预编译二进制、Docker 等多种方式


## 二、HAPI 解决的核心痛点

### 2.1 三大真实痛点

**① 内网孤岛效应**
AI 助手再强大，一下班就等于"离线"。灵感来了，却找不到它。

**② 远程操作困难**
用手机 VPN 连回公司电脑再操作桌面，不仅卡顿，更无法流畅地进行 AI 对话（尤其是长篇流式输出）。

**③ 碎片时间浪费**
通勤路上、咖啡厅里，突然想调一段代码或跑个测试，却只能干等回到工位。

### 2.2 适用人群

- **通勤开发者**：想在地铁/公交上继续写代码
- **多项目并行开发者**：需要远程管理多个 AI 会话
- **内网受限用户**：公司 AI 工具被锁定在内网
- **AI 工具重度使用者**：同时使用 Claude Code、OpenCode、Codex 等多种工具


## 三、工作原理与核心组件

### 3.1 "三位一体"的轻巧架构

HAPI 由三个紧密配合的小模块组成，各司其职：

```
┌─────────────────────────────────────────────────────────────────┐
│                        HAPI 架构图                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────────┐ │
│  │   CLI       │────▶│    Hub      │◀────│   Web 客户端    │ │
│  │  (命令层)   │     │  (调度中心) │     │   (手机界面)    │ │
│  └─────────────┘     └─────────────┘     └─────────────────┘ │
│        │                    │                      │           │
│        ▼                    ▼                      ▼           │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────────┐ │
│  │  AI 编程    │     │  实时同步   │     │  移动端优化     │ │
│  │  助手进程   │     │  状态转发   │     │  Web 应用       │ │
│  └─────────────┘     └─────────────┘     └─────────────────┘ │
│                                                                 │
│  HAPI 不替代你的 AI 工具，而是充当一座"桥梁"          │
└─────────────────────────────────────────────────────────────────┘
```

**① CLI（命令层）** 
运行在你的开发机上，负责和你本地的 AI 进程"对话"，并把结果传出去。

**② Hub（调度中心）** 
一个轻量的中转站，负责保持 CLI 和手机端的连接，实时同步状态。

**③ Web 客户端（手机界面）** 
一个专为移动端优化的网页应用，让你在手机上也能获得类似原生 App 的流畅体验。

### 3.2 工作流程

1. **手机端操作** → 你在手机 Web 页面上点击或输入指令
2. **Hub 转发** → Hub 将指令实时同步给开发机上的 CLI
3. **CLI 执行** → CLI 调用本地的 AI 编程助手（如 Claude Code）执行任务
4. **结果回传** → AI 的回复通过 Hub 实时推送到手机屏幕

> 整个过程实时流式传输，体验流畅，数据不经过任何第三方服务器。


## 四、核心特性与能力

### 4.1 本地优先，数据安全

所有数据都保存在本地电脑，不上服务器。对于想要数据自主、自托管、最小设置的用户而言，HAPI 是很好的选择。

### 4.2 多 AI 后端支持

HAPI 支持多种流行的 AI 编程工具：
- **Claude Code**（Anthropic）
- **OpenAI Codex**
- **Gemini CLI**（Google）
- **OpenCode**
- 以及通过 HAPI CLI 包装的任何 AI 编程工具

### 4.3 远程会话控制

- **远程开启会话**：从手机直接启动一个新的 AI 编程会话
- **会话切换**：在多个会话之间无缝切换
- **实时同步**：会话上下文和进度保持同步
- **文件浏览**：远程查看项目文件
- **Git Diff 查看**：远程查看代码变更
- **待办清单跟踪**：实时跟踪会话进度

### 4.4 权限管理与审批

AI 执行需要权限的操作时，会通过手机推送审批请求：
- **远程批准/拒绝**：一键批准或拒绝 AI 的工具使用请求
- **实时响应**：无需回到电脑前即可处理 AI 的权限请求

### 4.5 无缝切换

在本地终端和远程手机上可以无缝切换，AI 会话的上下文和进度会保持同步，不会丢失。

### 4.6 Web 客户端/PWA/Telegram Mini App 支持

- **Web 客户端**：通过浏览器访问，无需安装 App
- **PWA**：可安装为手机桌面应用
- **Telegram Mini App**：在 Telegram 内直接使用

### 4.7 多用户协作

HAPI 支持多用户协作场景，多个开发者可以共享 AI 会话。


## 五、安装与配置

### 5.1 环境要求

- **Node.js** 环境（推荐 v18+）
- **Bun**（如果从源码构建）
- **AI 编程助手**：如 Claude CLI（用于 Claude Code 会话）
- **可选：公网服务器**（用于自建中继）

### 5.2 安装 HAPI

```bash
# 通过 npm 全局安装（推荐）
npm install -g @twsxtd/hapi --registry=https://registry.npmjs.org
```

国内用户可使用镜像源加速：
```bash
npm install -g @twsxtd/hapi --registry=https://registry.npmmirror.com
```

### 5.3 启动模式

HAPI 支持多种启动模式，可根据网络环境选择：

#### 模式一：中继模式（Relay Mode）—— 零配置外网访问

使用 HAPI 作者提供的公共中继服务器：
```bash
hapi hub --relay
```

> **注意**：作者的中继在海外，国内网络环境可能无法连接，会卡在 `Waiting for trusted TLS certificate...`

#### 模式二：自建 Hub（自托管模式）

在自己的服务器上启动 Hub，作为中继服务器：

```bash
# 在服务器上启动
export HAPI_LISTEN_HOST=0.0.0.0
export HAPI_PUBLIC_URL="http://你的公网IP:3006"
hapi hub
```

首次运行会打印 `CLI_API_TOKEN`，保存在 `~/.hapi/settings.json` 中。

#### 模式三：纯内网模式

不配置公网访问，仅在同一内网使用。

### 5.4 配置 Runner（执行端）

如果需要在服务器上作为执行端接入 Hub：

```bash
# 告诉 Runner Hub 的地址
export HAPI_API_URL="http://你的公网IP:3006"
# 交互式登录
hapi auth login
```

根据提示输入 `CLI_API_TOKEN` 完成认证。

### 5.5 进程管理（推荐 PM2）

```bash
# 安装 PM2
npm install -g pm2

# 启动 Hub
pm2 start hapi -- hub

# 启动 Runner（如果需要）
pm2 start hapi -- hub runner
```


## 六、使用方法

### 6.1 启动 AI 编程会话

在终端中使用 `hapi` 前缀启动 AI 编程工具：

```bash
hapi claude          # Claude Code
hapi codex           # OpenAI Codex
hapi opencode        # OpenCode
hapi gemini          # Gemini CLI
```

### 6.2 手机端访问

1. **通过 Web 客户端**：
   - 在手机浏览器中打开 Hub 的公网地址（如 `http://你的IP:3006`）
   - 输入 `CLI_API_TOKEN` 登录

2. **通过 Telegram Mini App**：
   - 配置 Telegram Bot Token
   - 在 Telegram 中直接操作

### 6.3 常用操作

| 操作 | 说明 |
|------|------|
| **查看会话** | 在手机端查看所有活跃的 AI 会话 |
| **发送指令** | 向 AI 发送文本指令，实时查看回复 |
| **切换会话** | 在多个项目会话间切换 |
| **审批权限** | 远程批准/拒绝 AI 的工具调用请求 |
| **查看文件** | 浏览远程项目文件 |
| **查看 Git Diff** | 查看代码变更 |


## 七、与 AstrBot 的集成

HAPI 可以与 AstrBot 深度集成，形成完整的远程 Vibe Coding 方案。

### 7.1 AstrBot 插件：hapi_connector

社区已开发 `astrbot_plugin_hapi_connector` 插件，让 AstrBot 成为 HAPI 的**消息入口**。

**工作流程**：
1. 用户在微信/QQ/Telegram 发送指令
2. AstrBot 通过 hapi_connector 插件转发给 HAPI
3. HAPI 调用本地的 AI 编程助手
4. 结果实时推送到聊天窗口

### 7.2 集成后的能力

- **远程指令控制**：通过聊天软件发送 `/hapi list`、`/hapi create` 等命令
- **实时通知推送**：AI 的回复实时推送到聊天窗口
- **权限审批**：在聊天窗口中远程批准 AI 的操作请求
- **多会话管理**：在不同聊天窗口中管理不同项目的 AI 会话


## 八、HAPI vs OpenClaw：如何选择？

当你想在内网引入 AI 辅助时，可能会遇到 HAPI 和 OpenClaw 两个选项，它们定位不同：

| 维度 | HAPI（远程控制） | OpenClaw（全能助手） |
|------|-----------------|---------------------|
| **核心定位** | "遥控器"：远程操作你已有的 AI 工具 | "机器人"：自主完成任务的 OS |
| **稳定性** | 极高：链路简单直接 | 中等：架构复杂 |
| **安全性** | 高：数据不离开内网 | 需警惕：曾报告安全风险 |
| **部署难度** | 极轻：几个命令即可启动 | 较重：需要 Node、Docker 等 |
| **权限范围** | 受控：只能做 AI 工具能做的事 | 宽泛：可能直接操作系统界面 |

**通俗理解**：
- 如果你只想**远程使用公司内网现有的、你信任的 AI 编程助手** → 选 HAPI
- 如果你需要的是一个能**自动处理邮件、操作多个软件**的自主 Agent → 选 OpenClaw


## 九、常见问题与注意事项

### 9.1 国内网络环境特别提示

- **中继连接问题**：`hapi hub --relay` 连接的是海外中继，国内可能无法连接
- **解决方案**：使用自建 Hub（`hapi hub` 不带 `--relay`）
- **公网服务器**：如有公网服务器，可将其作为中继站

### 9.2 端口冲突问题

- Hub 默认占用端口 3006
- 如需修改，通过环境变量 `HAPI_LISTEN_PORT` 设置
- Runner 使用的 `HAPI_API_URL` 不占用端口

### 9.3 Token 管理

- 首次运行 Hub 会生成 `CLI_API_TOKEN`，保存在 `~/.hapi/settings.json`
- 手机端登录和 Runner 认证都需要这个 Token
- **务必保存好**，丢失后需要重新生成

### 9.4 依赖要求

- HAPI 需要 Claude CLI 等 AI 后端才能工作
- 确保 AI 工具已正确安装并配置好 PATH

### 9.5 开源协议注意事项

- CLI 部分：MIT
- Server/Web 部分：LGPLv2
- LGPLv2 对闭源商业产品的集成有要求，需仔细评估


## 十、总结

HAPI 是一个**本地优先、轻量、安全的 AI 编程远程控制框架**。它通过 CLI + Hub + Web 的三位一体架构，让你能随时随地通过手机远程操控电脑上的 AI 编程助手。

它的核心价值在于：**把内网的 AI 编程能力，安全、稳定地"搬"到你的手机或任何能上网的设备上**。

**一句话总结**：HAPI 让你在任何地方都能继续你的"Vibe Coding"——不是通过 VPN 连回电脑操作桌面，而是通过一个专门为 AI 远程控制设计的轻量框架，获得流畅、安全的移动端 AI 编程体验。
