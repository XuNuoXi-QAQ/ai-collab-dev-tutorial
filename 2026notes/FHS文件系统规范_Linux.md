Linux 系统常用目录清单。


## 📂 根目录 (`/`) 下的核心目录

这是 Linux 系统的基础骨架，每个目录都有其特定用途。

| 英文 | 目录 | 中文含义 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **Binaries** | `/bin` | 二进制文件 | 存放系统最基本的用户命令（如 `ls`、`cp`、`cat`），所有用户均可执行。在部分现代发行版中，它是 `/usr/bin` 的符号链接。 |
| **System Binaries** | `/sbin` | 系统二进制文件 | 存放系统管理命令（如 `fdisk`、`reboot`、`ifconfig`），通常需要 root 权限才能执行。用于系统启动、修复和网络配置。 |
| **Editable Text Configuration (或 "et cetera")** | `/etc` | 可编辑的文本配置 | **系统级配置文件存放地**。几乎所有的系统服务（如网络、用户账户、软件启动项）的配置文件都在这里，是 Linux 最重要的目录之一。 |
| **Home Directories** | `/home` | 用户家目录 | 普通用户的个人文件夹所在地。每个用户有独立子目录（如 `/home/alice`），用于存放个人文档、配置和下载内容。 |
| **Root User's Home** | `/root` | 超级管理员家目录 | root 用户（系统最高权限账户）的个人文件夹。与 `/home` 分开是为了确保在系统维护或救援模式下依然可访问。 |
| **Libraries** | `/lib` | 基础库文件 | 存放 `/bin` 和 `/sbin` 下命令所依赖的**共享库文件**（类似于 Windows 的 `.dll`）以及内核模块。是系统启动和运行的最小依赖集。 |
| **Devices** | `/dev` | 设备文件 | 存放所有硬件设备的文件接口。例如 `/dev/sda` 代表第一块硬盘，`/dev/tty` 代表终端。这是 Linux **“一切皆文件”** 哲学的集中体现。 |
| **Process Information** | `/proc` | 进程信息 | **虚拟文件系统**，不占用硬盘空间。提供内核和进程运行时的动态信息，如 `/proc/cpuinfo`（CPU信息）、`/proc/meminfo`（内存信息）。 |
| **System** | `/sys` | 系统内核接口 | **虚拟文件系统**，用于与 Linux 内核交互，管理和暴露硬件设备、驱动及内核子系统的信息。是设备管理和电源管理的关键接口。 |
| **Temporary Files** | `/tmp` | 临时文件 | 存放应用程序运行期间产生的临时数据。系统重启时通常会被清空，任何用户都有读写权限。适合存放短期缓存。 |
| **Boot** | `/boot` | 启动文件 | 存放系统启动所需的文件，包括 Linux 内核（`vmlinuz`）、初始内存磁盘（`initrd.img`）和引导加载程序（如 GRUB）的配置。 |
| **Optional Add-on Software** | `/opt` | 可选附加软件 | 用于安装**第三方独立软件包**（非系统自带）。每个软件通常有自己的子目录（如 `/opt/deerflow`、`/opt/n8n`），便于独立管理。 |
| **Services** | `/srv` | 服务数据 | 存放本机提供的网络服务（如 HTTP、FTP）所产生的数据文件。例如 Web 服务的网页文件可放在 `/srv/www/`。 |
| **Mount** | `/mnt` | 挂载点（临时） | 系统管理员**临时挂载**其他文件系统（如另一块硬盘、网络存储）的传统挂载点。属于手动管理的位置。 |
| **Media** | `/media` | 挂载点（自动） | 可移动媒体设备（如 U 盘、光盘、移动硬盘）插入后，系统**自动挂载**的目录。由桌面环境和系统服务（如 udisks）自动管理。 |
| **Lost+Found** | `/lost+found` | 丢失与修复 | 在系统发生意外崩溃或文件系统损坏时，通过 `fsck` 工具找回的丢失文件会被存放在此。并非所有发行版都有此目录。 |
| **Run** | `/run` | 运行时数据 | 存放系统启动以来的**临时运行时数据**，如进程 PID 文件、系统守护进程的 socket（套接字）文件。使用 tmpfs（内存文件系统），重启后数据会清空。 |

## 📦 `/usr` (Unix System Resources) 下的子目录

`/usr` 是系统最主要的软件存放区，容量通常最大，包含了绝大多数用户级程序、库和文档。

| 缩写全称 (英文) | 目录 | 中文含义 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **User Binaries** | `/usr/bin` | 用户级命令 | 存放绝大多数用户可用的命令和程序（如 `vim`、`python`、`gcc`）。这是 Linux 系统命令最集中的目录。 |
| **System Binaries** | `/usr/sbin` | 额外的系统命令 | 存放系统管理员使用的、非启动必需的额外管理工具（如 `nginx`、`mysqld`）。 |
| **Libraries** | `/usr/lib` | 软件库文件 | 存放 `/usr/bin` 和 `/usr/sbin` 下程序所依赖的共享库、以及许多软件的数据文件和配置文件。 |
| **Local** | `/usr/local` | 本机安装软件 | 用于存放**本机管理员**自行编译安装的软件，或者从源代码手动安装的程序，避免与系统包管理器（如 apt、yum）管理的软件发生冲突。其内部目录结构与 `/usr` 类似（如 `/usr/local/bin`、`/usr/local/lib`）。 |
| **Shared Data** | `/usr/share` | 架构无关的共享数据 | 存放与硬件平台无关的只读数据文件，包括**文档**（`/usr/share/doc`）、**帮助手册**（`/usr/share/man`）、**图标**、**字体**、**时区信息**等，供所有用户使用。 |
| **Include Files** | `/usr/include` | 头文件 | 存放 C、C++ 等编程语言的标准**头文件**（如 `stdio.h`、`stdlib.h`）。在编译软件时，编译器需要从这里引用所需的 API 定义。 |
| **Source Code** | `/usr/src` | 源代码 | 存放 Linux 内核源代码或其他开源软件源代码的位置。通常只有内核源码包或调试符号会放在这里。 |
| **Games** | `/usr/games` | 游戏程序 | 存放系统预装的游戏程序（如 `fortune` 或 `nethack`）。在现代发行版中此目录较少使用。 |

## 📝 `/var` (Variable Data) 下的子目录

`/var` 用于存放系统运行期间会**频繁变化**的数据，如日志、缓存、队列等。

| 缩写全称 (英文) | 目录 | 中文含义 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **Log Files** | `/var/log` | 日志文件 | **系统日志主目录**。几乎所有系统服务、应用和内核的日志文件都存放在这里（如 `/var/log/syslog`、`/var/log/nginx/access.log`），是排查问题的重要依据。 |
| **Cache** | `/var/cache` | 缓存数据 | 存放应用程序的**可重新生成的缓存**。例如软件包管理器（apt、yum）的下载包缓存，可以在需要时被安全删除以释放空间。 |
| **Spool** | `/var/spool` | 假脱机队列 | 存放等待处理的任务队列，如待发送的邮件（`/var/spool/mail`）、待打印的文档（`/var/spool/cups`）或定时任务（`/var/spool/cron`）。 |
| **Lock Files** | `/var/lock` | 锁文件 | 存放用于防止多个进程同时访问同一资源的**锁文件**。通常位于 `/run/lock`（符号链接）。 |
| **Run** | `/var/run` | 运行时数据 | 存放系统启动以来的临时 PID 文件和 socket。通常是指向 `/run` 的符号链接。 |
| **Temporary (persistent)** | `/var/tmp` | 持久化临时文件 | 存放程序产生的临时文件，但与 `/tmp` 不同，系统重启后**不会自动清空**。适合存放需要跨重启持久化的临时数据。 |
| **Mail** | `/var/mail` | 邮件队列 | 系统用户的电子邮件存储目录（传统位置）。部分系统已迁移至 `/var/spool/mail`。 |
| **State** | `/var/lib` | 状态数据 | 存放应用程序的**状态信息**和持久化数据，如数据库文件（`/var/lib/mysql`、`/var/lib/postgresql`）、包管理器的状态数据库等。 |

## 🔍 `/proc` (Process Information) 下的关键虚拟文件

这些不是真实文件，而是内核暴露给用户空间的“查看窗口”，通过读取它们可以了解系统运行状态。

| 缩写全称 (英文) | 目录/文件 | 中文含义 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **CPU Info** | `/proc/cpuinfo` | CPU 信息 | 显示 CPU 型号、核心数、频率、缓存大小等信息。 |
| **Memory Info** | `/proc/meminfo` | 内存信息 | 显示系统物理内存、交换分区的使用情况（总内存、可用内存、缓存、缓冲区等）。 |
| **Uptime** | `/proc/uptime` | 系统运行时间 | 显示系统自启动以来的总秒数，以及空闲时间的秒数。 |
| **Load Average** | `/proc/loadavg` | 系统负载 | 显示系统过去 1、5、15 分钟的平均负载（正在运行或等待运行的进程数）。 |
| **Filesystems** | `/proc/filesystems` | 已支持的文件系统 | 列出内核当前支持的文件系统类型（如 ext4、xfs、ntfs）。 |
| **Mounts** | `/proc/mounts` | 当前挂载信息 | 显示系统当前所有挂载的文件系统（相当于 `mount` 命令的输出）。 |
| **Networking** | `/proc/net/` | 网络统计 | 包含网络协议（TCP/UDP）的统计信息、路由表、ARP 缓存等，如 `/proc/net/tcp`。 |
| **Process Directories** | `/proc/[PID]/` | 进程详情 | 每个正在运行的进程都有自己的目录（以 PID 命名），包含其环境变量、打开的文件、内存映射等信息。 |
